1.2 用例规则 ​

  • 文件名以 test_开头 或 _test结尾；
  • 类名以 Test 开头，并且不能带有 __init__方法；

            x = "hello"
            assert hasattr(x, 'check')
            
        def test_three(self):
            a = "hello"
            b = "hello world"
            assert a in b

1.3 用例执行 ​

  • -m:标记用例
    • pytest -m login
  • -q:简单打印，只打印测试用例的执行结果
    • pytest -q start.py
  • -s:详细打印
    • pytest -s start.py
  • -x:遇到错误时停止测试
    • pytest start.py -x
  • --maxfail=num:当用例错误个数达到指定数量时，停止测试
    • pytest start.py --maxfail=1
  • -k:匹配用例名称
    • pytest -s -k http start.py 执行测试用例名称包含 http 的所有用例
    • pytest -s -k "not http" start.py 根据用例名称排出某些用例
    • pytest -s -k "method or weibo" start.py 同时匹配不同的用例名称
shell
pytest

pytest 脚本名称.py

pytest -v 08_mark.py::TestClass::test_method

pytest 08_mark.py::TestClass::test_method

断言异常 ​

异常断言时：type 异常类型不需要加引号，value 需要转为 str

def test_zero_division_long():
    with pytest.raises(ZeroDivisionError) as excinfo:
        1 / 0

    # 断言异常类型 type
    assert excinfo.type == ZeroDivisionError
    # 断言异常 value 值
    assert "division by zero" in str(excinfo.value)

拓展一：match，正则异常断言 ​

  • match 以正则表达式与异常字符串表示形式是否匹配
  • 该方法只断言 value， 不断言 type
  • match 方法的regexp参数与 re.search 函数匹配，因此 match='zero' 也可以使用
python
def test_zero_division_long():
    with pytest.raises(ZeroDivisionError, match=".*zero.*") as excinfo:
        1 / 0

def test_zero_division_long():
    with pytest.raises(ZeroDivisionError, match="zero") as excinfo:
        1 / 0

二、setup 和 teardown ​

  • 模块级别： setup_module、teardown_module
  • 函数级别（不在类中）： setup_function、teardown_function
  • 类级别（在类中）： setup_class、teardown_class
  • 方法级别（在类中）： setup_method、teardown_method
  • 方法细化级别： setup、teardown
python
import pytest


def setup_module():
    print("=====模块级-前置，只执行一次=====")
    
def teardown_module():
    print("=====模块级-后置，只执行一次=====")
    
def setup_function():
    print("===函数级-前置，每个测试函数都执行===")
    
def teardown_function():
    print("===函数级-后置，每个测试函数都执行====")

def test_one():
    print("one")
    
def test_two():
    print("two")

class TestCase():
    def setup_class(self):
        print("====类级-前置，每个测试类都执行====")

    def teardown_class(self):
        print("====类级-后置，每个测试类都执行====")

    def setup_method(self):
        print("====方法级-前置，每个测试方法都执行====")

    def teardown_method(self):
        print("====方法级-后置，每个测试方法都执行====")

    def setup(self):
        print("=类里面每个用例执行前都会执行setup=")

    def teardown(self):
        print("=类里面每个用例结束后都会执行teardown=")

    def test_three(self):
        print("three")

    def test_four(self):
        print("four")

if __name__ == '__main__':
    pytest.main(["-q", "-s", "-ra", "setup_teardown.py"])

3.1 参数列表 ​

python

def test():

2
3
  • scope: 作用域，默认为 function，还有 class、module、package、session四个
  • autouse：默认为 False，需要手动调用 fixtrue
    • 如果是 True，所有作用域内的测试用例都会自动调用该 fixture
  • name：默认为 装饰器的名称，同一个模块需要不同的 name

WARNING

session 的作用域：是整个测试会话，即开始执行 pytest 到结束测试

3.2 调用 fixtrue ​

  • 在类上面加上 @pytest.mark.usefixtures(), 表示这个类中的所在测试用例都会调用该 fixture
  • 如果有多个 @pytest.mark.usefixtures(), 则从下往上执行，即距离类近的先执行
  • 多个 fixture 参数，先执行前面的，后执行后面的
  • 如果 fixture 有返回值，则必须用传参的方式
python
import pytest

# 调用方式一
@pytest.fixture
def login():
    print("输入账号，密码先登录")

def test_s1(login):
    print("用例 1：登录之后其它动作 111")

def test_s2():  # 不传 login
    print("用例 2：不需要登录，操作 222")


# 调用方式二
@pytest.fixture
def login2():
    print("please输入账号，密码先登录")

@pytest.mark.usefixtures("login2", "login")
def test_s11():
    print("用例 11：登录之后其它动作 111")


# 调用方式三
@pytest.fixture(autouse=True)
def login3():
    print("====auto===")

# 不是test开头，加了装饰器也不会执行fixture
@pytest.mark.usefixtures("login2")
def loginss():
    print(123)

if __name__ == '__main__':
    pytest.main(["-q", "-s", "-ra"])

3.3 fixtrue 实例化顺序 ​

session > package > module > class > function 在 fixture_A 里面依赖的 fixture_B 优先实例化，然后再 fixture_A 实例化 自动使用（autouse=True）的 fixture 将在显式使用（传参或装饰器）的 fixture 之前实例化

python
import pytest

order = []

@pytest.fixture(scope="session")
def s1():
    order.append("s1")

@pytest.fixture(scope="module")
def m1():
    order.append("m1")

@pytest.fixture
def f1(f3, a1):
    # 先实例化f3, 再实例化a1, 最后实例化f1
    order.append("f1")
    assert f3 == 123

@pytest.fixture
def f3():
    order.append("f3")
    a = 123
    yield a

@pytest.fixture
def a1():
    order.append("a1")

@pytest.fixture
def f2():
    order.append("f2")

def test_order(f1, m1, f2, s1):
    # 按照 fixture 执行顺序为：s1 -> m1 -> f3 -> a1 -> f1 -> f2
    assert order == ["s1", "m1", "f3", "a1", "f1", "f2"]

3.5 yield 实现 teardown ​

用 yield 关键字开启 teardown 操作

WARNING

  • 如果 yield 之前的代码抛出异常，则不会执行 yield 之后的代码
  • 如果测试用例抛出异常，yield 后面的代码正常执行
python
import pytest

@pytest.fixture(scope="session")
def open():
    # 会话前置操作setup
    print("===打开浏览器===")
    test = "测试变量是否返回"
    yield test
    # 会话后置操作teardown
    print("==关闭浏览器==")

@pytest.fixture
def login(open):
    # 方法级别前置操作setup
    print(f"输入账号，密码先登录{open}")
    name = "==我是账号=="
    pwd = "==我是密码=="
    age = "==我是年龄=="
    # 返回变量
    yield name, pwd, age
    # 方法级别后置操作teardown
    print("登录成功")

def test_s1(login):
    print("==用例1==")
    # 返回的是一个元组
    print(login)
    # 分别赋值给不同变量
    name, pwd, age = login
    print(name, pwd, age)
    assert "账号" in name
    assert "密码" in pwd
    assert "年龄" in age

def test_s2(login):
    print("==用例2==")
    print(login)

3.7 addfinalizer 终结函数 ​

如果 request.addfinalizer() 前面的代码，即setup部分已经抛出异常了， 则不会执行 request.addfinalizer() 的teardown内容 可以声明多个终结函数并调用

python
@pytest.fixture(scope="module")
def test_addfinalizer(request):
    # 前置操作setup
    print("==再次打开浏览器==")
    test = "test_addfinalizer"

    def fin():
        # 后置操作teardown
        print("==再次关闭浏览器==")

    request.addfinalizer(fin)
    # 返回前置操作的变量
    return test


def test_anthor(test_addfinalizer):
    print("==最新用例==", test_addfinalizer)

四、conftest.py ​

conftest.py 专门存放 fixture 的配置文件

4.1 配置注意事项 ​

  • pytest 会默认读取 conftest.py 里面的所有 fixture
  • conftest.py 文件名称是固定的，不能改动
  • conftest.py 只对同一个 package 下的所有测试用例生效
  • 不同目录可以有自己的 conftest.py, 一个项目中可以有多个 conftest.py
  • 测试用例文件中不需要手动 import conftest.py, pytest 会自动查找

5.1 @pytest.mark.skip ​

跳过执行测试用例, 可选参数 reason: 跳过的原因, 会在执行结果中打印

知识点

  • @pytest.mark.skip 可以加在函数上, 类上, 类方法上
  • 如果加在类上面, 类里面的所有测试用例都不会执行

import pytest

@pytest.mark.skip(reason="不执行该用例！！因为没写好！！")

    @pytest.mark.skip(reason="不想执行")
    def test_2(self):
        print("%% 我是类测试用例2222 %%")

@pytest.mark.skip(reason="类也可以跳过不执行")

    def test_1(self):
        print("%% 不会执行 %%")

5.4 跳过标记 ​

@skipmark
class TestSkip_Mark(object):

    def test_function(self):
        print("测试标记")

        print("测试标记")

@skipmark
def test_skip():
    print("测试标记")
